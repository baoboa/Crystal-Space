C.2 Release Notes from 1.4 to 1.4.1

This section documents changes between versions 1.4 and 1.4.1 of Crystal Space.

Fixes that apply to tickets in Trac:

  • Ticket #737: Projects created by ‘jamtemplate’/‘createproject.sh’ now take MSVC_VERSION into account when linking against CEL libraries.
  • Ticket #803: Don't use a deprecated ‘libpng’ function.
  • Ticket #805: Fix problem where csTinyXmlDocument::Parse(iDataBuffer*) overload did not ensure that string passed to Parse(const char*) overload was null-terminated.
  • Ticket #810: Fix OpenGL context-related build error on Snow Leopard by taking advantage of API available since 10.3 rather than employing hack needed by earlier versions.
  • Ticket #811: Fix cannot convert size_t to UInt32 compilation error on Snow Leopard where size_t is now 64-bit by default rather than 32-bit.
  • Ticket #812: Fix leak reported in oggstream2. ov_clear must be called when the ogg stream is no more needed according to vorbis documentation. It seems as a result leaks reported by openal have disappeared. patch by agy.
  • Ticket #813:

    Fix problem where version number was appended to the executable name at ‘jam install’ time on Windows without regard for file extension (i.e. ‘foobar.exe-1.9’) rather than being inserted just before the extension (i.e. ‘foobar-1.9.exe’).

    Fix problem where symbolic-link creation failed on Windows at ‘jam install’ time when linking non-versioned to versioned executable name.

  • Ticket #817: Update CS repository URL to latest format specified by SourceForge.
  • Ticket #895: Cherry picked doc updates. Many thanks to darkenor for taking the time to make the submission.

Other changes:

  • Removal of special autoresume code when unpausing on Mac OS X as multiple consecutive NSApplicationDidResignActiveNotification messages may be sent causing the app to permanently freeze.
  • Set CS_CONFIGDIR for VFS.
  • Change Bullet configure test to look for (apparently) new-style BulletCollision -lLinearMath as well as previous -lbulletcollision and -lbulletmath.
  • Moved ‘Python.h’ _DEBUG hack into it's own file.
  • Fixed release mode compile errors in cspython.
  • Fixed usage of m4_default in call to CS_CHECK_PROG_VERSION to work with the now stable interface tested on ‘autoconf’ 2.66 and 2.67
  • Relocated POSIX dl-based ‘libs/csutil/unix/loadlib.cpp’ to ‘libs/csutil/generic’ since it should be portable to various POSIX platforms.

    Genericized ‘libs/csutil/generic/loadlib.cpp’ to allow platform-specific file extensions (‘.csplugin’/‘.so’) to be specified at compile time.

    Retired the Mac OS X specific dynamic module loading facilities in favor of ‘libs/csutil/generic/loadlib.cpp’. This is possible since Mac OS X supports POSIX ‘dl’ facilities, unlike ancestors NextStep, OpenStep, Rhapsody from which the OS X port arose. In addition to reducing Mac OS X specific implementation, this also resolves Apple deprecation warnings from the Darwin module-loading API.

  • Fix wrongly placed break.
  • Fix a wrong data size validation check. Spotted by Lirrec.
  • Remove redundancy.
  • 1.4 requires CEGUI 0.5 but is known to work with 0.6. 1.4 will not compile with CEGUI 0.7.x.

    We don't want bug reports about this on such an old release and the work involved in changes to 1.4 to now support 0.7 is not really worth it.

    Added test for versions greater than 0.6 of CEGUI, so it fails the 1.4 ‘configure’ test.

  • Add arm support.
  • Disable Java build.
  • Removed ‘csdemo’ from the default build.

    This is due to the ‘demodata.zip’ file no longer being available. Code has been left in place for review and learning purposes. Removed ‘csdemo’ from ‘startme’ application list. Updated docs to indicate the above.

  • Documented options for running ‘phystut’ with preferred physics plug-in more explanatory.
